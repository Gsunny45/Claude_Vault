## Decision (Mars-ratified, 2026-05-29)

1. **Canonical keyboard injection vault = `android-ai-keyboard-harness\Hermetic_A-Z_Vault`.**
   It already holds the prompt/context/provider/knowledge corpus, already lives in the
   project folder, and is already consumed by the wired `ObsidianBridge` path. The
   DEC-0003/0006 boundary holds — this is content beside the code, not a Claude_Vault note set.

2. **`Documents\Hermes_Phone_Vault` stays the separate device-ops vault.** It is NOT the
   keyboard vault and was never meant to be. Its DP/DT device schema is correct for its
   purpose. No move, no merge.

3. **Vault_Skills design `Hermes-Phone-Vault.md` (dual-purpose merge) is PARKED — do not build.**
   Superseded by this decision. The name's similarity to the device-ops folder was the core
   of the "mass vault confusion." Recommend renaming the design doc to
   `_PARKED_Hermes-Phone-Vault (dual-purpose, superseded by DEC-0007).md` to kill the collision.

4. **`Keyboard_AI_Vault.md` injection-wiring spec (2026-05-11) is IMPLEMENTED.** Its
   "What's Broken / Unwired" table is now stale — `{{file.path}}`, `{{file.tags}}`,
   `{{file.tags|none}}`, `{{vault.path}}`, body preservation all ship in current code
   (CteEngine.kt ~L744-787, ObsidianBridge.kt readFrontmatter/FrontmatterResult).

## Context

[[DEC-0003]] (2026-04-19) iceboxed the keyboard, sending it to a new non-vault folder `Documents\agentA-Z\` with its own repo, and moved [[tasks/TSK-0003]] to `cancelled`. That folder was never scaffolded.

Reality diverged. The project resumed as an active FlorisBoard fork at `C:\Users\MarsBase\Documents\android-ai-keyboard-harness` (app "HermeticA-Z", pkg `dev.patrickgold.florisboard.vault.debug`), with a UI facelift built `BUILD SUCCESSFUL` and installed on the Moto g 5G as recently as 2026-05-27→28 (`inbox/hermetic_facelift_handoff.md`, `build-final.log`). The 2026-05-28 alignment pass ([[knowledge/KNW-0024]]) surfaced the gap.

## Alternatives

1. **Keep docs frozen, treat the harness as off-the-books.** Rejected — guarantees the same doc-vs-reality drift recurs every session.
2. **Re-icebox the live project to match the docs.** Rejected — it is actively built and deployed; freezing it would be fiction.
3. **Amend DEC-0003: un-icebox, unify the name to the real harness, keep the not-a-vault boundary.** Accepted.

## Decision

- The keyboard is an **active project**, not iceboxed.
- Canonical name/path: **`android-ai-keyboard-harness`** at `Documents\android-ai-keyboard-harness\`. The `agentA-Z` name and `Documents\agentA-Z\` path are retired (never existed on disk).
- **What carries over unchanged from DEC-0003:** the keyboard is NOT a vault; its code, `res/`, and build artifacts do not belong inside Claude_Vault; keyboard internals are not pulled into active vault briefings. Keyboard *knowledge/handoff* records (KNW-0008, KNW-0024, the handoff in `inbox/`, the device note) are allowed as boundary pointers.

## Consequences

- CLAUDE.md "Explicitly not part of this system" entry rewritten: keyboard is active at the harness path, boundary preserved.
- [[tasks/TSK-0003]] → `status: in_progress`, retitled, `new_home`/`project` repointed to the harness, `uniceboxed: 2026-05-28` added; icebox banner replaced with a revival banner.
- DEC-0003 frontmatter `amended_by: [DEC-0004]`; its icebox bullet annotated.
- Repo hygiene: four misrouted keyboard build logs (`build-debug.log`, `build-debug2.log`, `build-debug3.log`, `build-final.log`) moved out of Claude_Vault root into `_system/logs/keyboard-build/` (kept, not deleted — training-data rule, vault rule 9). Long-term home is the harness repo, not the vault.

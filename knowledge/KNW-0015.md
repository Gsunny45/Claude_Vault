# OpenRouter MCP Server

## Location

`Claude_Vault/_scripts/openrouter_mcp/openrouter_mcp.py`

## Purpose

The internet-facing AI layer of the orchestration stack. One API key routes to 300+ models (Claude, Gemini, Llama, Mistral, Qwen, etc.) via OpenRouter. Includes automatic fallback chains for resilience — if one model is down, the next takes over.

## Tools (6 total)

| Tool | Type | Purpose |
|------|------|---------|
| openrouter_chat | Write | Single-turn completion to any model |
| openrouter_chat_multi | Write | Multi-turn with conversation history |
| openrouter_chat_fallback | Write | Resilient chat — auto-fallback across model chain |
| openrouter_list_models | Read | Search/browse 300+ models with pricing |
| openrouter_model_info | Read | Detailed specs for a specific model |
| openrouter_check_credits | Read | Account balance and usage monitoring |

## Integration Points

- **n8n**: HTTP Request node can call OpenRouter directly OR this MCP routes requests
- **RAG pipeline**: Vault context + question → openrouter_chat_fallback → answer back to vault
- **Agent routing**: Different agents (Pope Bot, Agent0) can use different models via this gateway
- **Local fallback**: Chain can include local models if Ollama endpoint is proxied through OpenRouter-compatible API

## Transport

stdio (local process, for Claude Desktop/Code/Cowork integration)

## Dependencies

Python: mcp, httpx, pydantic

## Setup

1. `pip install mcp httpx pydantic`
2. Set `OPENROUTER_API_KEY` environment variable
3. Add to Claude Desktop config or Claude Code via `claude mcp add`
4. Full instructions in `_scripts/openrouter_mcp/README.md`

## Default Fallback Chain

gemini-flash → gemini-pro → claude-3.5-sonnet → llama-3.1-70b

# Skill Assessment for Resilient Local System Build

## Tier 1 — Core Build Skills

| Skill | Why |
|-------|-----|
| **mcp-builder** | Every bridge in the architecture is an MCP server. Obsidian REST API, OpenRouter, local models, agent endpoints. |
| **engineering:architecture** | Structured decisions with tradeoffs. Stops re-debating solved problems. |
| **engineering:system-design** | The layered system (WSL/Ubuntu/Kali/PowerShell + agents + Obsidian) is a system design problem. |
| **skill-creator** | Build custom skills for RAG pipelines, OpenRouter routing, agent deployment. Persistent knowledge across sessions. |
| **engineering:debug** | Structured debugging instead of thrashing. |
| **schedule** | Automated polling, health checks, vault sync. |

## Tier 2 — Support Skills

| Skill | Why |
|-------|-----|
| **engineering:documentation** | Agent runbooks, system docs, the breadcrumbs that keep getting lost. |
| **productivity:memory-management** | Two-tier memory. Session-to-session continuity. |

## Tier 3 — Not Needed Now

design:*, docx, pptx, xlsx, pdf, internal-comms, UX copy — output production, not infrastructure.

## Custom Skills to Build (via skill-creator + mcp-builder)

1. **OpenRouter routing** — model selection, fallback chains, cost tracking
2. **RAG pipeline** — vault search → context assembly → prompt → response routing
3. **Agent deployment** — stand up new purpose agents (Pope Bot, Agent0, OpenClaw, Hermes)
4. **Local model management** — Ollama/llama.cpp health, model swaps, benchmarks

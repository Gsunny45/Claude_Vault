## Extraction procedure (safe, non-destructive)

Order matters. Do these one at a time, verify after each, commit between steps.

### 1. Claude_on_Claude — extract nested repo

It already has its own remote and an "Initial commit" on 2026-04-17. Easiest extraction:

```bash
# From Claude_Vault root
cd Claude_on_Claude
git log --oneline   # confirm remote is github.com/Gsunny45/Claude_on_Claude.git
git status          # capture any uncommitted changes first

# Move the folder out intact (preserves .git)
cd ..
mv Claude_on_Claude "$HOME/Documents/Claude_on_Claude"   # WSL path; or use File Explorer from Windows side

# From Claude_Vault, clean up the dirty index entry
git rm -r --cached Claude_on_Claude   # removes index entry, leaves working tree alone (it's already moved)
git commit -m "extract Claude_on_Claude to sibling repo"
```

### 2. llm-orchestrator — initialize new repo, then extract

This is bigger (frontend/backend/training_data). Do NOT use `git filter-branch` — overkill. Just move it, init a fresh repo, and push.

```bash
# Capture current state in Claude_Vault's history first (so the content is preserved in history if anything goes wrong)
cd ~/Documents/Claude_Vault
git add llm-orchestrator/
git commit -m "snapshot: llm-orchestrator before extraction"

# Move the folder
mv llm-orchestrator ~/Documents/llm-orchestrator

# Init new repo
cd ~/Documents/llm-orchestrator
git init
git add .
git commit -m "initial commit — extracted from Claude_Vault"
# Then create github.com/Gsunny45/llm-orchestrator, add remote, push

# Back in Claude_Vault, remove the folder from tracking
cd ~/Documents/Claude_Vault
git rm -r llm-orchestrator
git commit -m "remove llm-orchestrator (extracted to own repo)"
```

## Guardrails to prevent repeat drift

- **One git remote per working directory.** Run `git remote -v` before committing if in doubt.
- **No nested `.git` directories.** Obsidian-git's backup auto-commit will silently cross them and produce confusing histories.
- **Claude Code WSL sessions** should use `realpath` / `pwd -P` at the start of any multi-step task to pin the working directory — not a `C:\...` string.
- **Training-data paths are gold** (per Mars): `sessions/`, `inbox/`, `_system/logs/`, `llm-orchestrator/training_data/`. Never prune these; when extracting llm-orchestrator, make absolutely sure `training_data/` goes with it.
- **Android shell logs (Termux/Alpine/Kali) and PowerShell WSL/Kali logs** are training corpus when they land here. Same rule — do not prune.

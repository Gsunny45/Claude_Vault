## Summary

Local-Network-Hub is the user's production AI workflow coordination system at `C:\Users\MarsBase\Documents\Local-Network-Hub`. It's an Obsidian vault designed as a persistent workspace where humans and AI agents collaborate through standardized note structure, task queue, session logging, and handoff mechanisms.

## Architecture

Flat structure (max 1 level nesting):

| Folder | Purpose |
|--------|---------|
| `00-System/Sessions/` | One .md per AI work session — persistent memory |
| `01-Projects/` | Active projects with status/priority frontmatter |
| `02-Areas/` | Ongoing responsibilities |
| `03-Inbox/` | Quick capture — default location for new unsorted notes |
| `04-Archive/` | Completed/abandoned work (never delete) |
| `Templates/` | Session-Log, Project, Decision, Inbox templates |

## Key Files

- `CLAUDE.md` — master context document (Claude Code reads automatically)
- `AGENTS.md` — provider-agnostic version for non-Claude agents
- `_handoff.md` — current session state snapshot, updated by each AI session
- `Dashboard.md` — live Dataview queries showing system state

## Frontmatter Schema (shared "API")

```yaml
type: session | project | decision | inbox | area | pattern | reference
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: active | paused | complete | abandoned | unsorted
priority: high | medium | low
tags: [array]
```

## REST API

- Port 27124 (HTTPS) with API key auth
- Port 27123 (insecure, disabled)
- Enables external systems to read/write vault programmatically

## Claude Code Integration

`.claude/settings.local.json` restricts writes to 00-System, 01-Projects, 03-Inbox, _handoff.md, Dashboard.md. Templates are write-protected.

- `cog_foundation_v2.py` (1,452 lines) — multi-AI orchestration with 4-phase reasoning (plan, critique, execute, output), COG Bus for task routing
- `cog_server.py` — Flask REST server for workflow execution

## Design Philosophy

Human + AI partnership model: session logs create continuous memory, frontmatter is the machine-readable API, _handoff.md is the state bridge between sessions, Git provides audit trail. The vault is designed to scale to multiple coordinated vaults.

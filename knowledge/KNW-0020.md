# Handoff Protocol

Full specification lives in the Claude_on_Claude repo:
`Claude_on_Claude/docs/HANDOFF-PROTOCOL.md`

## Key Concepts

Every task carries a **handoff packet**: source_node, destination_node, executor_agent, priority, payload, and persist_to. No task exists without a clear routing path.

## Routing Summary

| Task Type | Primary Node | Fallback |
|-----------|-------------|----------|
| Infrastructure build | Claude Code (WSL) | Claude Desktop |
| Git operations | Claude Code (WSL) | PowerShell |
| System maintenance | Claude Desktop + user | Claude Code |
| Architecture decisions | Claude Desktop | — |
| Workflow routing | n8n | PowerShell scripts |
| Autonomous tasks | OpenClaw | Claude Code |
| Fast inference | Ollama → Gemini → Groq | OpenRouter free |
| Deep reasoning | Claude Desktop Pro | OpenRouter paid |
| Memory persistence | Claude Vault | Local-Network-Hub |

## Return-to-Vault Rule

Every completed task persists results to the appropriate vault location:
- Knowledge → `knowledge/KNW-NNNN.md`
- Decisions → `decisions/DEC-NNNN.md`
- Task status → `tasks/TSK-NNNN.md`
- Code → `Claude_on_Claude` repo (git commit)
- Integration changes → `KNW-0011`

## Cross-Environment Patterns

WSL ↔ Windows: OAuth always needs Windows browser. File access via `/mnt/c/`. Git works natively in WSL.

Local ↔ VPS (planned): Deploy via git push or scp. Trigger via n8n webhooks. Memory sync via Obsidian REST API.

# n8n Workflow: Vault RAG Loop

## Purpose

Obsidian note → n8n reads it → Gemini answers using vault context → answer written back to vault.
This is the first working integration that proves the full stack.

## Architecture

```
[Schedule Trigger: every 5 min]
        │
        ▼
[HTTP Request: GET inbox notes from Local-Network-Hub REST API]
        │
        ▼
[IF: note has status=unsorted and tag=rag-question]
        │
        ▼
[HTTP Request: GET related vault notes for context (search endpoint)]
        │
        ▼
[Gemini Pro: answer question using context]
        │
        ▼
[HTTP Request: PATCH original note with answer + set status=complete]
```

## n8n Build Instructions (Node by Node)

### Prerequisites
- Local-Network-Hub vault open in Obsidian
- REST API plugin active (port 27124)
- REST API key (find in Obsidian settings → Local REST API → API Key)
- Gemini API key (from Google AI Studio)

### Node 1: Schedule Trigger
- Type: **Schedule Trigger**
- Interval: Every 5 minutes
- This kicks off the pipeline on a loop

### Node 2: Read Inbox Notes
- Type: **HTTP Request**
- Method: `GET`
- URL: `https://localhost:27124/vault/03-Inbox/`
- Headers:
  - `Authorization`: `Bearer YOUR_REST_API_KEY`
- Options: SSL → turn OFF certificate verification (it's a self-signed cert)
- This returns a list of notes in the inbox folder

### Node 3: Split Into Items
- Type: **Split In Batches** (or Item Lists → Split Out)
- This processes each inbox note individually

### Node 4: Read Each Note Content
- Type: **HTTP Request**
- Method: `GET`
- URL: `https://localhost:27124/vault/03-Inbox/{{ $json.filename }}`
- Headers:
  - `Authorization`: `Bearer YOUR_REST_API_KEY`
  - `Accept`: `application/vnd.olrapi.note+json`
- Options: SSL → turn OFF certificate verification
- Returns the note content + frontmatter as JSON

### Node 5: Filter — Is This a RAG Question?
- Type: **IF**
- Condition: `{{ $json.frontmatter.tags }}` contains `rag-question`
- AND: `{{ $json.frontmatter.status }}` equals `unsorted`
- Only proceed if it's an unprocessed question

### Node 6: Search Vault for Context
- Type: **HTTP Request**
- Method: `POST`
- URL: `https://localhost:27124/search/simple/`
- Headers:
  - `Authorization`: `Bearer YOUR_REST_API_KEY`
  - `Content-Type`: `application/json`
- Body (JSON):
```json
{
  "query": "{{ $json.content }}",
  "contextLength": 300
}
```
- Options: SSL → turn OFF certificate verification
- Returns matching vault passages as context for the AI

### Node 7: Gemini Pro — Answer the Question
- Type: **Google Gemini Chat Model** (or HTTP Request to Gemini API)
- If using HTTP Request:
  - Method: `POST`
  - URL: `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-pro:generateContent?key=YOUR_GEMINI_KEY`
  - Body:
```json
{
  "contents": [{
    "parts": [{
      "text": "You are a research assistant with access to the user's knowledge vault. Answer the question using ONLY the provided context. If the context doesn't contain the answer, say so.\n\n## Context from vault:\n{{ $node['Search Vault for Context'].json.results }}\n\n## Question:\n{{ $node['Read Each Note Content'].json.content }}"
    }]
  }]
}
```

### Node 8: Write Answer Back to Vault
- Type: **HTTP Request**
- Method: `PATCH`
- URL: `https://localhost:27124/vault/03-Inbox/{{ $node['Read Each Note Content'].json.filename }}`
- Headers:
  - `Authorization`: `Bearer YOUR_REST_API_KEY`
  - `Content-Type`: `application/vnd.olrapi.note+json`
- Body:
```json
{
  "frontmatter": {
    "status": "complete",
    "answered_by": "gemini-pro",
    "answered_date": "{{ $now.format('yyyy-MM-dd') }}"
  },
  "content": "{{ $node['Read Each Note Content'].json.content }}\n\n---\n## AI Answer (Gemini Pro)\n\n{{ $node['Gemini Pro'].json.candidates[0].content.parts[0].text }}"
}
```
- Options: SSL → turn OFF certificate verification

## Testing

1. Create a note in Local-Network-Hub `03-Inbox/` with this exact format (template below in vault)
2. Wait for schedule trigger (or click "Test Workflow" in n8n)
3. Check the note — it should now have an `## AI Answer` section appended and status=complete

## Failure Modes
- REST API key wrong → 401 on all HTTP nodes
- SSL cert error → turn off verification in every HTTP node
- Gemini key wrong → 400 from Google
- No matching context → Gemini will say "I don't have enough context" (this is correct behavior)

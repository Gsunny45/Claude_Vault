# Obsidian Plugins for Claude/LLM Integration

Research date: 2026-04-14. Focused on what adds value beyond what's already installed (Dataview, Templater, REST API, Smart Connections, Kanban, Periodic Notes).

## Priority 1: Claudian (7.7K stars, MIT)

Direct Claude Code integration — makes the vault the agent's working directory. Supports custom instruction files (AGENTS.md, CLAUDE.md) and MCP servers. Can invoke the REST API via bash within agent context.

**Why it matters:** Bridges the gap between Obsidian and the LLM Orchestrator. Claude can read/write vault notes directly, invoke the orchestrator API, and maintain state across sessions.

**Install:** Community plugins → search "Claudian"

## Priority 2: Copilot for Obsidian Plus (6.7K stars, free + $10/mo)

Mature agent autonomy with long-term memory and autonomous tool-calling. Complements Smart Connections with active agentic behavior rather than passive RAG. No pre-built embeddings required.

**Why it matters:** Adds an in-vault AI agent that can use the knowledge base proactively — drafting, summarizing, querying across notes.

## Priority 3: Text Generator (1.9K stars, MIT)

Flexible multi-provider text generation (Claude, OpenAI, Google, local models). Handles routine content generation within existing Templater workflows. Lightweight.

**Why it matters:** Quick inline generation without leaving Obsidian. Can point at the orchestrator's API for multi-model routing.

## Emerging Opportunity: Obsidian MCP Server

The 24,700+ repo MCP ecosystem is standardizing. Building an Obsidian MCP server would expose vault search and decision-ledger as tools any LLM can call. This would let the orchestrator backend directly query the vault during RAG-augmented chat.

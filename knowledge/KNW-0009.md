## Summary

Curated inventory of open-source repositories relevant to the agentA-Z Keyboard project. All are Android-native, local-first capable, actively maintained, and support GGUF models. Filtered for forkability (MIT/Apache/GPLv3).

## Repos by Category

### Local LLM Inference (run Qwen2.5-Coder-1.5B on-device)

| Repo | License | Key Feature | Integration Path |
|------|---------|-------------|-----------------|
| SmolChat-Android | Apache-2.0 | JNI + llama.cpp, APK ready | Best pure GGUF runner for exact model |
| ChatterUI | Open | React Native + llama.cpp background service | Can run as persistent server |
| MLC-LLM | Apache-2.0 | GPU/NPU accel via TVM | Highest perf on supported SoCs |
| mllm | — | In-app Go server, excellent streaming | Good for Qwen specifically |
| llama.cpp (Termux) | MIT | Build from source, full control | Server mode at localhost:8080 |

### Trigger / Template Layer (Templater-style on Android)

| Repo | License | Key Feature | Integration Path |
|------|---------|-------------|-----------------|
| **SwiftSlate** | MIT | Accessibility Service + custom triggers + OpenAI-compatible endpoint | **#1 starting point** — already does what Templater does on Android |
| textexpander_android | GPLv3 | Espanso YAML parser → system-wide expansion | Fork and swap static replace with LLM call |
| expander | — | Lightweight text expansion | Simple trigger reference |

### Voice → Polish (Typeless-style)

| Repo | License | Key Feature | Integration Path |
|------|---------|-------------|-----------------|
| whisper-to-input | GPLv3 | Full Android IME with mic button | Supports self-hosted Whisper backend |
| whisper.cpp | MIT | Core STT engine + Android examples | examples/whisper.android for native integration |
| OpenTypeless | MIT | Desktop multi-provider STT+LLM | Inspiration only (not Android-native) |

### Base Keyboards (build-from-scratch path)

| Repo | License | Key Feature | Integration Path |
|------|---------|-------------|-----------------|
| **FlorisBoard** | Apache-2.0 | Modern Kotlin IME, extension store, NLP hooks | **Primary IME base** for full custom keyboard |
| AnySoftKeyboard | Apache-2.0 | Battle-tested, multi-language | Alternative if FlorisBoard proves too complex |
| KeyboardGPT | — | Xposed module, custom LLM prompts | Requires root (LSPosed), limited localhost |

## Recommended Build Approaches (ranked by speed-to-prototype)

| Option | Base | Effort | Root? | Best For |
|--------|------|--------|-------|----------|
| A: Accessibility + Local API | SwiftSlate + llama.cpp server | 1-2 weeks | No | Fastest path to triggers + CoT/ToT |
| B: Full Custom IME | FlorisBoard + whisper-to-input | 3-4 weeks | No | Voice + typing + templates in one app |
| C: Xposed Module | KeyboardGPT | 1 week | Yes | AI on any existing keyboard |

**Winner for R&D phase:** Option A (SwiftSlate) for immediate triggers, transition to Option B (FlorisBoard) for full product.

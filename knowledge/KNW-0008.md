## Summary

agentA-Z is an Android AI keyboard system with two design paths. Design 1 is cloud-hybrid (ships first), Design 2 is fully local (open-source future). Both share the same 5-layer architecture flowing top-to-bottom.

## Architecture (5 Layers)

### Layer 1 — Input (IME)
FlorisBoard fork. Typing + voice input (Whisper.cpp + Typeless-style polish). Trigger detection: keyword prefixes (`//claude`, `/tot`, `/mcp`, `/skill`), grammar-error hooks, pause detection (>2s), swipe gestures, long-press.

### Layer 2 — Orchestration (JSON Engine)
`skills.json` config file loaded on IME start. Jinja2/Kotlin-rendered templates with variable expansion (`{{userInput}}`, `{{cot_analysis}}`). Context engine retrieves from local vault (SQLite-vec + MiniLM embeddings) and injects at system prompt level. Per-app tone profiles (WhatsApp = casual, Gmail = formal, IDE = code).

### Layer 3 — Reasoning (CoT + ToT)
CoT: Step-by-step analysis for conversational/dictation use cases. Low latency, streaming.
ToT: Multi-branch generation (2-3 options scored on accuracy/speed/maintainability) for analytical/document work. Longer latency, higher quality.
Grammar Reconfig: AI suggestions injected into the candidate bar on-the-fly when triggered by spelling errors, pauses, or explicit commands.

### Layer 4 — AI Providers
Design 1: Claude API, MCP servers (.mcp.json), SSH/remote shell, Claude Code, Claude Desktop.
Design 2: Ollama/MLX/ONNX/GGUF local models, self-hosted MCP servers, open-source skills repo.

### Layer 5 — Output
Rich suggestions bar (CleverType-style: Option 1 / Option 2 / Apply Template). Actions: insert, replace, copy to clipboard, save to vault as new skill, trigger next skill in chain.

## JSON Config Schema (v1.0)

Top-level structure:
- `schemaVersion` — config format version
- `vaultMetadata` — API keys, default model, encryption, max context tokens
- `skills[]` — array of skill objects, each with: id, name, trigger patterns, inputTypes, orchestration (CoT/ToT steps with promptTemplates and outputVars), toneTemplates, grammarReconfigHook, outputActions, testCases
- `globalTemplates` — reusable prompt fragments
- `contextEngine` — vector DB path, embedding model

Key skill: `meta_skill_builder` — the "skill that builds skills." Takes task description + examples, runs CoT analysis then ToT branching, outputs new SKILL.md or JSON skill config. This is the self-improving core.

### Diagnosis
Something else on Mars's machine is already listening on 127.0.0.1:5678. Candidates:
1. n8n-desktop app (cloud trial) still running in tray/background
2. Another n8n process from earlier install
3. A stray node process

### Resolution — n8n was NEVER down, was a native install all along

`Get-NetTCPConnection -LocalPort 5678` showed wslrelay.exe (PID 17444). wslrelay.exe just forwards ports from WSL distros to Windows. So the real listener was inside WSL.

Ran via Windows-MCP PowerShell tool (Claude executing directly, not Mars pasting):

```
wsl -d Ubuntu -- ss -tlnp | grep 5678
  LISTEN 0 4096 0.0.0.0:5678 0.0.0.0:*

wsl -d Ubuntu -- ps aux | grep -iE 'n8n|node'
  gitdevop 695 node /usr/local/bin/n8n           # ← the real n8n
  gitdevop 1298 node .../n8n/task-runner/dist/start.js

Invoke-WebRequest http://localhost:5678/healthz → 200 OK
Invoke-WebRequest http://localhost:5678/ → 200 OK

ps -p 695: CPU 0.7%, MEM 5.9%, uptime 41:49 — healthy, not pegged
```

**n8n is a npm-installed native service in Ubuntu WSL, owned by user `gitdevop`, PID 695.** It's healthy and has been running for 41+ minutes. The Docker `n8n_local` container was always a redundant leftover — it never needed to run.

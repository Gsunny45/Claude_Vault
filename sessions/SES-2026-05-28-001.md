# SES-2026-05-28-001 — Hermes Android Auth Investigation

## Goal
Investigate why the Hermes Agent Android app (Nous Research) was inaccessible despite an active Plus subscription. Unblock Mars from using the subscription before the July 1 Nous Chat shutdown.

## Device
- Samsung Galaxy Note 20 Ultra (SM-N986U), Android 13
- ADB via WiFi (Windows-MCP PowerShell)
- App: com.nousresearch.hermesagent v0.13.112 (Alpha)

## Key Findings

### Week-long blocker solved
The Corr3xt URL field was blank. Mars had been trying to find `corr3xt` as a separate service to install. It is not self-hosted — it is simply `https://portal.nousresearch.com`. Entering that URL and tapping Refresh activates the email/Google sign-in buttons.

### Subscription status confirmed (via portal.nousresearch.com)
- Plan: Plus ($20/month)
- Credits: $21.74 of $22 remaining
- Renews: 2026-06-21
- API credits: $0.00 (separate bucket — subscription credits ≠ API credits)
- Subscription credits only usable via: Nous Chat (browser) or Hermes Agent (Android app)

### Root cause of Android sign-in failure
The app generates this OAuth URL:
```
https://portal.nousresearch.com/oauth/start
  ?method=email&provider=email&client=hermes-android
  &callback_contract=v1
  &redirect_uri=hermesagent%3A%2F%2Fauth%2Fcallback
  &state=4cfb10ad-eda9-4d62-b22a-40c7c38ea53e
```
`/oauth/start` returns **HTTP 404** for `client=hermes-android`. The endpoint `/oauth/auth` exists (returns 429) but `/oauth/start` is not deployed. This is a **server-side Nous Research bug** — no client-side fix possible.

### ADB techniques used
- `uiautomator dump` → pull XML → parse bounds for precise tap coordinates
- `input tap`, `input keyevent`, `input text` via ADB shell
- Triple-tap + KEYCODE_DEL spam to clear corrupted URL field
- `monkey -p com.nousresearch.hermesagent 1` to launch app

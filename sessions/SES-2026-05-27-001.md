## What Remains Broken (User-Visible)

### 1. Launcher icon not appearing correctly
- **Root cause:** Adaptive icon XMLs updated, but PNG mipmaps (`mipmap-mdpi/` through `mipmap-xxxhdpi/`) are NOT regenerated. Android uses cached or fallback icons.
- **Fix required:** Generate all 5 density PNGs from `ic_app_icon_debug_foreground.xml` and replace `res/mipmap-*/ic_launcher.png` and `ic_launcher_round.png`.
- **Tool:** Android Studio Asset Studio, or `rsvg-convert` / `inkscape` CLI ‚Üí 5 sizes (48/72/96/144/192dp).

### 2. Two missing startup screens
- **Status:** Unknown which screens. User stated "missing two startup screens to start with."
- **Candidates needing investigation:**
  - A branded Hermetic splash/onboarding flow beyond `HermeticSetupWizard`
  - An AI status/provider health screen shown on first launch
  - A CTE intro or capability overview screen
- **Next step:** Ask user one question: "Which two screens were showing before that are now missing ‚Äî can you describe what was on them?"
- **Note:** `HermeticSetupWizard` IS present and functional. The missing screens are something else.

### 3. HomeScreen has no Hermetic branding
- `HomeScreen.kt` is unmodified stock FlorisBoard settings list.
- CTE entry is a single bottom item ‚Äî no AI status card, no provider health, no Hermetic identity.
- **Fix:** Add `HermeticStatusCard` composable at top showing: active provider name, budget state, quick CTE action.

### 4. Keyboard surface still stock green (not auto-themed)
- `cyberdeck_night.json`, `hermetic_night.json` exist and are registered in `extension.json`.
- NOT auto-selected on install ‚Äî user must manually navigate Settings ‚Üí Theme after every fresh install.
- **Fix:** Wire `HermeticSetupWizard` FINISH step ‚Üí `ThemeManager.setActiveTheme("hermetic_night")`.

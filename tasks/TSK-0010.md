# TSK-0010 — Daily Workflow Nerve

> The workflow of the day. Every morning, the system wakes up, reads the vault, triages tasks, and prepares a briefing. This is the heartbeat.

## Objective

Build an automated daily workflow that:
1. Reads the vault state (open tasks, recent changes, drift)
2. Uses a local LLM or free cloud model to triage and summarize
3. Writes a daily briefing note to the vault
4. Flags anything urgent

This replaces the manual "read _briefing.md and _drift_report.md" step from the session protocol. The system does it for you.

## Architecture

```
Schedule Trigger (n8n, 6:00 AM daily)
    │
    ▼
Read vault tasks/ (HTTP Request → REST API :27124)
    │
    ▼
Read vault _system/_drift_report.md
    │
    ▼
Read vault knowledge/ (recent changes only)
    │
    ▼
Summarize + Triage (Ollama local OR Gemini Flash)
    │
    ├─ Priority tasks for today
    ├─ Stale items needing attention
    ├─ System health status
    └─ Suggested focus areas
    │
    ▼
Write daily note → vault sessions/daily/YYYY-MM-DD.md
    │
    ▼
(Optional) Send summary to agentmail or notification
```

## Acceptance Criteria

- [ ] n8n workflow built (or standalone Python script if n8n not ready)
- [ ] Reads at least: tasks/, _system/_drift_report.md, recent inbox/
- [ ] Produces a structured daily briefing with: task triage, health check, focus areas
- [ ] Writes briefing to sessions/daily/ with proper frontmatter
- [ ] Falls back: n8n → Python script → manual (always works offline)
- [ ] Tested with both Ollama (local) and Gemini Flash (cloud)
- [ ] Documented in KNW-NNNN

## Agent Instructions

**For n8n (primary — when Hostinger is live):**
Build as a workflow. Schedule Trigger → multiple HTTP Requests → Code node (summarize) → HTTP Request (write back).

**For Claude Code (WSL) — offline fallback:**
Build as `daily_nerve.py` in Claude_on_Claude/local/. Uses Ollama at localhost:11434 and vault REST API at localhost:27124. Run via cron or manual invocation.

**For Gemini Flash:**
When vault content is passed to you, produce a structured triage. Format: priority tasks, stale items, health flags, suggested focus. Keep it under 500 words.

**For Ollama (local):**
Same as Gemini but expect slower processing. Use the smaller Qwen2.5-Coder model for speed, or Llama 3.1 8B for quality.

# TSK-0009 — Force Multiplication RAG Index

> 1,600+ files of accumulated human knowledge sit in FM v1. Agents can't use what they can't find. This task makes that knowledge searchable.

## Objective

Create a structured index of Force Multiplication v1 that any agent can query. This feeds the RAG pipeline — when an agent needs context about a topic Mars has researched, it should find it in FM v1 through the vault REST API.

## Why Gemini Flash is Ideal

Gemini Flash has a 1M token context window and 250 free requests/day. It can ingest large chunks of the FM v1 vault and produce structured summaries that smaller models can then use for retrieval.

## Acceptance Criteria

- [ ] Scan FM v1 folder structure (top-level categories, file counts per folder)
- [ ] Identify the 20 most important/central files (by links, size, or naming)
- [ ] Create KNW-0021: FM v1 folder map with category descriptions
- [ ] Create KNW-0022: FM v1 key entry points and how to navigate
- [ ] Create a topic index: map of keywords → file paths for RAG retrieval
- [ ] Store topic index in Claude_on_Claude/local/fm-v1-index.json
- [ ] Test: given a question, can the index find the right FM v1 note?

## Agent Instructions

**For Gemini Flash (primary — 1M context):**
Ingest FM v1 folder listings and key files. Produce structured summaries and topic mappings. This is your strongest use case — large context, fast processing.

**For local LLM (Ollama):**
Process individual FM v1 files for keyword extraction. Build the topic→file mapping incrementally.

**For Claude Code (WSL):**
Scan the filesystem at `/mnt/c/Users/MarsBase/Documents/Force Multiplication v1/`. Generate file listings, word counts, link graphs. Build the JSON index.

# TSK-0006 — Local LLM Bones

> Local models are the bones — the structural foundation that holds everything up when the internet goes dark. This task hardens the local inference layer.

## Objective

Get Ollama running as an always-on service in WSL, select the right models for this hardware (i5-1335U, 16GB RAM, no GPU), and wire it as the first fallback in every chain.

## Hardware Reality Check

- CPU: i5-1335U, 10 cores, 1.30 GHz base — expect 5-15 tok/s
- RAM: 16GB total, ~12-13GB usable — ONE 8B model at a time max
- GPU: Iris Xe 128MB — useless for inference
- Storage: 134GB free — can afford 2-3 models (4-8GB each)

## Acceptance Criteria

- [ ] `ollama serve` runs as background service (auto-starts with WSL)
- [ ] Primary model pulled: Llama 3.1 8B Q4_K_M (~4.7GB, best quality/speed for this CPU)
- [ ] Secondary model pulled: Qwen2.5-Coder 1.5B Q8 (~1.6GB, fast code tasks)
- [ ] Benchmark both: measure tok/s for a standard prompt, log results
- [ ] Ollama accessible from localhost:11434 confirmed
- [ ] Create systemd user service or .bashrc alias for auto-start
- [ ] Test: pipe a vault note through Ollama for summarization
- [ ] Update KNW-0011 integration registry: Ollama = ACTIVE
- [ ] Update fallback-chains/default-chain.json with confirmed local endpoint

## Agent Instructions

**For Claude Code (WSL) — primary executor:**
```bash
# Start Ollama
ollama serve &

# Pull models
ollama pull llama3.1:8b-instruct-q4_K_M
ollama pull qwen2.5-coder:1.5b-instruct-q8_0

# Benchmark
time curl -s http://localhost:11434/api/generate -d '{"model":"llama3.1:8b-instruct-q4_K_M","prompt":"Summarize the purpose of a multi-agent orchestration system in 3 sentences.","stream":false}' | python3 -c "import sys,json; r=json.load(sys.stdin); print(f'Tokens: {r.get(\"eval_count\",0)}, Time: {r.get(\"eval_duration\",0)/1e9:.1f}s, tok/s: {r.get(\"eval_count\",0)/(r.get(\"eval_duration\",1)/1e9):.1f}')"
```

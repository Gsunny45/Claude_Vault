# TSK-0005 — Obsidian Blood Flow

> Obsidian is the blood of this system. Every agent reads from it, every result writes back to it. This task makes that flow reliable.

## Objective

Build and test a complete REST API pipeline so ANY agent (local LLM, free cloud, Claude Code) can read, write, and search the vault over HTTP. This is the connective tissue that makes the nerve system work.

## Why This Matters

Without vault connectivity, every agent is isolated. Knowledge dies in each session. The blood stops flowing. This task ensures:
- Local LLMs (Ollama) can query vault context before answering
- Free cloud agents (Gemini, Groq) can persist their findings
- n8n workflows can read/write vault notes
- OpenClaw agents on Hostinger can sync memory back

## Acceptance Criteria

- [ ] REST API confirmed on :27124 with auth key
- [ ] curl test: read a note (GET /vault/knowledge/KNW-0010.md)
- [ ] curl test: create a note (PUT /vault/inbox/test-write.md)
- [ ] curl test: search notes (POST /search/simple/ with query)
- [ ] curl test: list folder (GET /vault/tasks/)
- [ ] Python helper script: `vault_api.py` with read/write/search functions
- [ ] Document all endpoints in KNW-0021
- [ ] Test from WSL (localhost) AND document how VPS will access (Cloudflare Tunnel)

## Agent Instructions

This task can be delegated to any agent with HTTP capabilities:

**For Gemini Flash / Groq / Free models:**
Read KNW-0011 for current API status. The REST API is at https://127.0.0.1:27124. Auth is Bearer token. SSL verification must be disabled (self-signed cert). Write your findings to this task file's Progress section.

**For Claude Code (WSL):**
Run curl tests directly. Build vault_api.py. Commit to Claude_on_Claude/local/.

**For Ollama (local):**
You can test connectivity by having your output piped through a script that calls the vault API.
